14 - JavaScript modularisieren

Code aufteilen mit export und import

Lernziele dieser Lektion

  • Verstehen, warum grosse JS-Dateien problematisch werden
  • Einfache ES-Module erstellen und verwenden
  • Named Exports und Default Exports unterscheiden
  • Code in sinnvolle Teilbereiche aufteilen
  • Eine kleine App-Struktur mit mehreren Dateien bauen

Das Problem mit einer grossen Datei

  • schwer lesbar
  • schwer testbar
  • schwer wiederverwendbar
  • mehr Konflikte bei Teamarbeit

Je mehr Funktionen in main.js landen, desto unübersichtlicher wird das Projekt.

Was ist ein Modul?

Ein Modul ist eine eigene JavaScript-Datei mit klarer Verantwortung.

  • stellt Funktionen oder Werte bereit
  • kapselt Logik
  • kann in anderen Dateien importiert werden

Typische Aufteilung


js/
  main.js
  math.js
  dom.js
  api.js
  utils/
    format.js

Jede Datei übernimmt eine klar abgegrenzte Aufgabe.

Modul im HTML aktivieren
  • ohne type="module" funktionieren import und export nicht
  • Browser behandelt die Datei dann als ES-Modul

Named Export


function add(a, b) {
    return a + b;
}

function subtract(a, b) {
    return a - b;
}

export { add, subtract };

Mehrere Funktionen können aus derselben Datei exportiert werden.

Named Import


import { add, subtract } from "./math.js";

console.log(add(4, 2));
console.log(subtract(4, 2));

Die Namen müssen mit dem Export übereinstimmen.

Import mit Alias


import { add as plus } from "./math.js";

console.log(plus(5, 3));

Mit as kann ein importierter Name lokal umbenannt werden.

Default Export


export default function formatPrice(value) {
    return `CHF ${value.toFixed(2)}`;
}
                

import formatPrice from "./formatPrice.js";

Pro Datei gibt es höchstens einen Default Export.

Named vs. Default

Named Export mehrere Exporte pro Datei möglich
Default Export genau ein Hauptexport pro Datei
Praxis oft Named Exports für Hilfsfunktionen

Dateipfade beim Import


import { renderUser } from "./ui/renderUser.js";
import { API_URL } from "../config.js";
  • ./ = aktueller Ordner
  • ../ = ein Ordner höher
  • Dateiendung .js im Browser angeben

Beispiel: Kleine Aufteilung


js/
  main.js
  students.js
  dom.js
                

// students.js
export const students = ["Mia", "Noah", "Lea"];

// dom.js
export function renderList(items) { /* ... */ }

// main.js
import { students } from "./students.js";
import { renderList } from "./dom.js";

Worauf soll man achten?

  • Dateien klein und thematisch klar halten
  • keine wilden Mischungen aus DOM, Logik und Daten
  • sprechende Dateinamen verwenden
  • Imports möglichst oben in der Datei sammeln

Ein sinnvoller Startpunkt

main.js ist oft die Datei, welche alles zusammenführt.

  • andere Module importieren
  • Event Listener registrieren
  • Initialisierung starten

Häufige Fehler

  • type="module" im HTML vergessen
  • falscher relativer Pfad
  • .js im Import vergessen
  • Named Export als Default importieren oder umgekehrt

Definition of Done

  • Die App besteht aus mehreren JS-Dateien
  • Funktionen werden sauber exportiert und importiert
  • Das HTML lädt den Einstieg per Modul-Script
  • Jede Datei hat eine klare Aufgabe

Arbeitsauftrag

  1. Projekt mit type="module" aufsetzen
  2. Hilfsfunktionen in eigene Dateien verschieben
  3. Imports und Exports korrekt verbinden
  4. Code testen und Struktur kurz reflektieren
