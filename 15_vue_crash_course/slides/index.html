Warum jetzt noch Vue?

  • Damit könnt ihr besser verstehen, was ein Framework abnimmt
  • Vue ist kein Ersatz für JavaScript, sondern baut darauf auf

Was ist Vue?

Vue ist ein JavaScript-Framework für interaktive Benutzeroberflächen.

  • verbindet Daten mit HTML
  • aktualisiert die Oberfläche automatisch
  • hilft beim Strukturieren grösserer Frontends

Wann hilft ein Framework?

  • wenn viele UI-Zustände gleichzeitig gepflegt werden
  • wenn dieselben Daten an mehreren Stellen erscheinen
  • wenn Komponenten wiederverwendbar sein sollen
  • wenn die App mit der Zeit wächst

    Was kommt in echten Vue-Projekten dazu?

    • Komponenten statt einer grossen Datei
    • props für Datenübergabe
    • computed für abgeleitete Werte
    • Lifecycle-Hooks für Laden und Initialisieren
    • Single File Components mit .vue-Dateien

    Ausblick: Vue-Ökosystem

    • Vite für Entwicklung und Build-Prozess
    • Vue Router für mehrere Seitenansichten
    • Pinia für globalen State
    • Komponentenbibliotheken für grössere UIs

    Wichtige Einordnung

    • Frameworks lösen nicht automatisch schlechte Logik
    • Saubere JavaScript-Grundlagen bleiben entscheidend
    • Wer JS versteht, lernt Vue deutlich schneller

    Definition of Done

    • Sie können grob erklären, was Vue.js ist
    • Sie erkennen data, methods und mount in einer einfachen Vue-App
    • Sie kennen v-model, v-for und v-if auf Einsteigerniveau
    • Sie wissen, dass Vue ein möglicher nächster Schritt nach Vanilla JS ist
