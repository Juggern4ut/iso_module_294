Aufgabenblatt: Notizen-Verwaltungsapplikation (JavaScript)

Allgemeine Beschreibung

Im Rahmen dieses Moduls sollen Sie eine Notizen-Verwaltungsapplikation (Notes App) entwickeln. Ziel ist es, grundlegende Konzepte moderner JavaScript-Entwicklung praktisch anzuwenden, insbesondere Modularisierung, Zustandsverwaltung sowie Interaktion mit einer API.

Die Applikation wird bewertet und ist als Einzelarbeit einzureichen.

Technische Informationen

  • Das Backend welches die Notizen verwaltet befindet sich auf der folgenden URL: http://notes.iso.lmeier.ch
  • Die Notizen befinden sich unter den folgenden Endpoints
    • GET /api/collections/notes/records zum Laden der Notizen
    • POST /api/collections/notes/records zum Erstellen einer Notiz
    • DELETE /api/collections/notes/records/:id zum Löschen der Notiz mit der Id :id
    • PATCH /api/collections/notes/records/:id zum Bearbeiten der Notiz mit der Id :id
  • Die Notizen erlauben die folgenden Felder
    • title String (Optional)
    • content String (Optional)
    • color String (Optional)
    • user String [UserID] (Optional)
    • done Boolean (Optional)
  • Die Benutzer befinden sich unter den folgenden Endpoints
    • GET /api/collections/users/records zum Laden aller User
    • PATCH /api/collections/users/records/:id zum Updaten des Users mit der Id :id (Authentifizierung benötigt)

Funktionale Anforderungen

Ihre Applikation muss folgende Funktionalitäten erfüllen:

1. Notizenverwaltung

  • Laden der bereits vorhandenen Notizen über die im Unterricht besprochene API.
  • Darstellen der folgenden Informationen pro Notiz:
    • Titel
    • Inhalt
    • Autor
    • Erstellungsdatum
    • Status (done)
  • Erstellen einer neuen Notiz
  • Bearbeiten einer bestehenden Notiz
  • Löschen einer bestehenden Notiz

2. Filter- und Suchfunktion

  • Implementierung eines Filters zur Suche nach Notizen
  • Die Suche darf lokal gehandhabt werden

3. Sortierung

Die Notizen müssen sortierbar sein nach:

  • Erstellungsdatum
    • Aufsteigend
    • Absteigend
  • Alphabetisch (Titel oder Inhalt)

4. Statusverwaltung

  • Jede Notiz muss als "abgeschlossen" (API-Feld: done) markiert werden können

5. Authentifizierung

  • Implementierung einer Login- und Logout-Funktion
  • Das Login-Formular darf nur angezeigt werden, wenn der Benutzer nicht eingeloggt ist
  • Die Notizenverwaltung darf nur nach einem erfolgreichen Login zugänglich sein

Technische Anforderungen

  • Verwendung von ES-Modulen (saubere Aufteilung des Codes in mehrere Dateien)
  • Vermeidung von unnötigen Code-Wiederholungen
  • Saubere und verständliche Struktur des Codes
  • Ausreichende Kommentierung des Codes

Bonuspunkte

Für die folgenden zusätzlichen Features können Bonuspunkte vergeben werden:

  • Ansprechendes und benutzerfreundliches Design (HTML/CSS)
  • Erweiterter Filter:
    • Filtern von Notizen nach dem Status "done"
  • Passwort ändern:
    • Implementierung eines Formulars zum Ändern des Benutzerpassworts
    • Verwendung eines PATCH-Requests an folgende URL: /api/collections/users/records/:id
    • Zu übermittelnde Felder:
      • oldPassword (das aktuelle Passwort des eingeloggten Users)
      • password (das neue Passwort des eingeloggten Users)
      • passwordConfirm (Wiederholung des neuen Passwortes des eingeloggten Users)

Abgabe

  • Der Code darf als ZIP-Datei via Email oder MatterMost oder als Link zu einem Git-Repo abgegeben werden
  • Der Code muss lauffähig sein
  • Optional: README-Datei mit kurzer Beschreibung der Umsetzung

Bewertungskriterien

Die Bewertung basiert auf:

  • Funktionalität (Erfüllung aller Anforderungen)
  • Codequalität (Struktur, Lesbarkeit, Modularität)
  • Benutzerfreundlichkeit
  • Erweiterungen (Bonuspunkte)
