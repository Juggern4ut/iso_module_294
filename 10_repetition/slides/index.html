10 - Repetition

Zusammenfassung zur Prüfungsvorbereitung

Ziele heute

  • Die bisherigen Themen strukturiert wiederholen
  • Zentrale Konzepte verknüpfen
  • Typische Prüfungsaufgaben vorbereiten
  • Unsicherheiten vor der Prüfung klären

Themenblock 1: Grundlagen

  • Variablen, Datentypen, Arrays, Objekte
  • Bedingungen mit if und Schleifen
  • Funktionen sauber strukturieren
  • Code lesen und erklären können

Themenblock 2: DOM und Events

  • Elemente mit querySelector(All) referenzieren
  • Mit addEventListener auf Benutzeraktionen reagieren
  • Formulardaten auslesen und validieren

Themenblock 3: Objekte und Klassen

  • Daten als Objekte modellieren
  • Methoden sinnvoll kapseln
  • Klassen für wiederverwendbare Strukturen einsetzen
  • Verantwortlichkeiten im Code trennen

Themenblock 4: Async und Fetch


async function loadData() {
    const response = await fetch(API_URL);
    if (!response.ok) {
        throw new Error(`HTTP ${response.status}`);
    }

    return response.json();
}

Wichtig: Response prüfen, Fehler behandeln, Resultat weiterverarbeiten.

Themenblock 5: Projektstruktur

  • Frontend und Backend klar trennen
  • Daten laden, rendern, verändern
  • CRUD-Ablauf verstehen: Create, Read, Update, Delete
  • Nach jeder Mutation den UI-Zustand synchron halten

Typische Prüfungsmuster

  • Fehlenden Event-Handler ergänzen
  • Eine Funktion debuggen oder erweitern
  • Einen Fetch-Request korrekt aufbauen oder debuggen

Prüfungs-Checkliste

  • Kann ich Code erklären, nicht nur abschreiben?
  • Kann ich Fehler in der Konsole lesen und einordnen?
  • Weiss ich, wann ich await nutzen soll?

Mini-Wiederholung: DOM + Fetch


submitButton.addEventListener("click", async () => {
    const title = titleInput.value.trim();
    if (!title) return;

    await createNote({ title: title });
    await loadNotes();
});

Hier treffen Eventhandling, Validierung und asynchrones Arbeiten zusammen.

Häufige Fehler

  • Selektoren stimmen nicht mit dem HTML überein
  • await fehlt bei asynchronen Aufrufen
  • Response wird nicht mit response.ok geprüft
  • Nach Änderungen wird die Ansicht nicht neu gerendert

Arbeitsauftrag

  1. Alle Themenblöke kurz durchgehen und offene Fragen markieren
  2. Die Aufgaben auf dem Blatt als Prüfungssimulation lösen
  3. Unsichere Stellen im eigenen Projekt gezielt wiederholen
  4. Eine persönliche Checkliste für die Prüfung erstellen
